## Як відкрити файли

### CSV файли (для Excel)

**Спосіб 1: Імпорт в Excel**
1. Відкрийте Excel
2. Файл → Відкрити → виберіть CSV файл
3. Якщо дані в одній колонці:
   - Виділіть колонку A
   - Дані → Текст по стовпцях
   - Виберіть "з роздільниками"
   - Поставте галочку на "крапка з комою" (;)
   - Готово

**Спосіб 2: Через Google Sheets**
1. Відкрийте Google Sheets (sheets.google.com)
2. Файл → Імпортувати → Завантажити
3. Виберіть CSV файл
4. Роздільник: крапка з комою (;)

### Markdown файл (для Word)

**Спосіб 1: Онлайн конвертер**
1. Відкрийте https://markdowntoword.com
2. Вставте вміст .md файлу
3. Завантажте .docx

**Спосіб 2: Через VS Code**
1. Встановіть розширення "Markdown All in One"
2. Відкрийте .md файл
3. Ctrl+Shift+P → "Markdown: Export to PDF"

**Спосіб 3: Просто відкрити**
- Файл .md можна відкрити в блокноті або VS Code
- Форматування буде видно як розмітка

## Як користуватися планом впровадження

### Колонки файлу 01_ПЛАН_ВПРОВАДЖЕННЯ.csv

| Колонка | Опис |
|---------|------|
| Рівень | 0-5, де 0 - найважливіше |
| Етап | Назва етапу впровадження |
| № задачі | Унікальний номер |
| Задача | Коротка назва |
| Опис | Детальний опис |
| Пріоритет | 1-Критичний ... 6-Опційно |
| Статус | ☐ Не розпочато / ⏳ В роботі / ✅ Виконано |
| Відповідальний | Хто виконує |
| Дата початку | Планова дата |
| Дата завершення | Фактична дата |
| Примітки | Додаткова інформація |

### Як оновлювати статус

1. Відкрийте файл в Excel
2. Знайдіть потрібну задачу
3. В колонці "Статус" змініть значення:
   - `☐ Не розпочато` → `⏳ В роботі` → `✅ Виконано`
4. Заповніть дати та примітки
5. Збережіть файл

## Рекомендований порядок роботи

### Крок 1: Вивчення документації
1. Прочитайте `04_ОПИС_СИСТЕМИ.md` - загальне розуміння
2. Перегляньте `01_ПЛАН_ВПРОВАДЖЕННЯ.csv` - що потрібно зробити
3. Перегляньте `06_ПРИКЛАДИ_ДАНИХ_ДОВІДНИКІВ.csv` - які дані потрібні

### Крок 2: Підготовка даних
1. Зберіть свої Excel файли з даними
2. Порівняйте зі структурою в `02_СТРУКТУРА_БАЗИ_ДАНИХ.csv`
3. Підготуйте дані у форматі, що відповідає структурі

### Крок 3: Початок розробки
1. Зареєструйтесь на GitHub, Vercel, Supabase
2. Розпочніть з задач рівня 0 (Фундамент)
3. Відмічайте виконання в плані

## Позначення пріоритетів

| Пріоритет | Значення | Коли робити |
|-----------|----------|-------------|
| 1 - Критичний | Без цього система не працює | Одразу |
| 2 - Високий | Основний функціонал | Після критичних |
| 3 - Середній | Важливий функціонал | Етап 2-3 |
| 4 - Нормальний | Стандартний функціонал | Етап 4-5 |
| 5 - Низький | Можна відкласти | Етап 6+ |
| 6 - Опційно | Бажано, але не обов'язково | За потреби |
